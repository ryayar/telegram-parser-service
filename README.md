# Telegram Parser Service

Сервис для мониторинга Telegram-групп и отправки уведомлений при появлении сообщений, совпадающих с фильтрами пользователя.

## Архитектура

Два независимых процесса, связанных через общую SQLite-базу:

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Bot API** | aiogram 3 | Управление группами/паттернами, отправка уведомлений |
| **Userbot** | Telethon | Вступление в группы, мониторинг сообщений, матчинг |

```
Пользователь ↔ Bot API ↔ SQLite ↔ Userbot ↔ Telegram-группы
```

## Возможности

- Добавление групп по ссылке / юзернейму — userbot автоматически вступает
- Два типа паттернов: точное слово и умный (учитывает опечатки, замены букв, кириллицу)
- Фильтрация паттернов по группам — для каждой группы можно выбрать свой набор слов
- Тихие часы с учётом часового пояса — уведомления приходят без звука
- Фото из сообщений прикрепляются к уведомлению
- История совпадений с пагинацией
- Дедупликация: одно объявление из нескольких групп = одно уведомление

## Структура проекта

```
telegram-parser-service/
├── shared/
│   ├── config.py          # Настройки (pydantic-settings, .env)
│   ├── database.py        # CRUD-функции, схема БД (SQLite + aiosqlite)
│   └── models.py          # Датаклассы: User, Group, Pattern, Match
│
├── bot_api/
│   ├── main.py            # Запуск бота, регистрация роутеров
│   ├── fsm_storage.py     # Персистентное хранилище FSM (SQLite)
│   ├── sender.py          # Фоновый процесс отправки уведомлений
│   ├── handlers/
│   │   ├── start.py       # /start, главное меню
│   │   ├── groups.py      # Управление группами
│   │   ├── group_detail.py# Настройка паттернов для группы
│   │   ├── patterns.py    # Управление паттернами
│   │   ├── settings.py    # Часовой пояс, тихие часы
│   │   ├── stats.py       # Статистика
│   │   └── history.py     # История совпадений
│   ├── keyboards/         # Inline-клавиатуры
│   └── states/            # FSM-состояния
│
├── userbot/
│   ├── main.py            # Запуск Telethon
│   ├── monitor.py         # Обработка входящих сообщений
│   ├── matcher.py         # Проверка паттернов, хеширование
│   ├── joiner.py          # Вступление в группы из pending_joins
│   └── captcha.py         # Прохождение emoji-капч
│
├── tests/                 # pytest, in-memory SQLite
├── data/                  # БД и скачанные медиафайлы (в .gitignore)
├── sessions/              # Сессия Telethon (в .gitignore)
├── Dockerfile.bot
├── Dockerfile.userbot
├── docker-compose.yml
└── deploy.sh
```

## Запуск

### Локально

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .

cp .env.example .env
# Заполни .env

# Терминал 1
python -m bot_api.main

# Терминал 2
python -m userbot.main
```

### Docker

См. [DOCKER.md](DOCKER.md) — там подробная инструкция, включая первичную авторизацию Telethon.

```bash
./deploy.sh
```

## Тесты

```bash
pip install -r requirements-dev.txt
pytest
```
