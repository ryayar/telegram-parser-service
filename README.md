# Telegram Parser Service

Сервис для мониторинга Telegram-групп (барахолок, объявлений) и отправки уведомлений пользователям при появлении сообщений, соответствующих их фильтрам.

## Описание

Система состоит из двух независимых компонентов, работающих параллельно и связанных через общую базу данных:

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Bot API** | aiogram | Взаимодействие с пользователями: регистрация, настройка фильтров, отправка уведомлений |
| **Userbot** | Telethon | Присутствие в группах, чтение сообщений, поиск совпадений |

### Bot API — функции

- Регистрация новых пользователей
- Добавление/удаление групп для мониторинга
- Управление ключевыми словами и паттернами
- Настройка тихих часов и часового пояса
- Просмотр статистики
- Фоновый процесс: опрос таблицы `matches` и отправка уведомлений

### Userbot — функции

- Чтение списка активных групп из БД
- Автоматическое вступление в новые группы по ссылке
- Прохождение простых капч (кнопки с emoji)
- Мониторинг входящих сообщений во всех группах
- Проверка сообщений по паттернам каждого пользователя
- Дедупликация (один текст из разных групп = одна запись)
- Запись совпадений в таблицу `matches`

## Жизненный цикл сообщения

```
1. Продавец пишет в группу "Продам iPhone 15 Pro"
                      │
                      ▼
2. Userbot получает сообщение (Telethon event)
                      │
                      ▼
3. Userbot ищет в БД: кто подписан на эту группу?
   → Находит: user_id = 123, 456, 789
                      │
                      ▼
4. Для каждого юзера проверяет его паттерны:
   • User 123: ищет "iphone"    → СОВПАЛО
   • User 456: ищет "macbook"   → не совпало
   • User 789: ищет "iphone 15" → СОВПАЛО
                      │
                      ▼
5. Проверка дубля (text_hash за последние 2 часа):
   • User 123: дубля нет → INSERT в matches
   • User 789: дубля нет → INSERT в matches
                      │
                      ▼
6. Bot API (фоновый процесс) видит новые записи:
   SELECT * FROM matches WHERE sent_at IS NULL
                      │
                      ▼
7. Bot API отправляет каждому юзеру:
   • Проверяет таймзону и тихие часы
   • User 123: сейчас 14:00 → обычное уведомление
   • User 789: сейчас 23:00 → silent=true
                      │
                      ▼
8. После отправки: UPDATE matches SET sent_at = now()
```

## Ограничения Telegram

| Параметр | Лимит | Решение |
|----------|-------|---------|
| Групп на аккаунт | 500 (free) / 1000 (premium) | Пул userbot-аккаунтов при масштабировании |
| Отправка сообщений | 30 msg/sec (Bot API) | Очередь с rate limiting |
| Вступление в группы | ~200 действий/24ч | Очередь pending_joins с задержками |

### Дедупликация

**Проблема:** Одно объявление публикуется в нескольких группах.

**Решение:**
1. Нормализация текста (lowercase, убираем лишние пробелы)
2. Вычисление MD5-хеша
3. Проверка: есть ли такой хеш для этого юзера за последние N часов
4. Дополнительно: fuzzy-matching (rapidfuzz) для похожих текстов с порогом 90%

> Дедупликация работает per-user. Если два юзера подписаны на одну группу, каждый получит своё уведомление.

### Тихие часы

1. Пользователь устанавливает часовой пояс через меню настроек
2. Пользователь устанавливает тихие часы (например, 23:00-08:00)
3. При отправке Bot API вычисляет текущее время пользователя
4. Если попадает в тихие часы — отправляет с `disable_notification=True`

## Запуск

### Разработка

```bash
# Установка зависимостей
python -m venv venv
source venv/bin/activate  # Linux/macOS
pip install -r requirements.txt

# Терминал 1 — Bot API
cd bot_api && python main.py

# Терминал 2 — Userbot
cd userbot && python main.py
```

### Docker Compose

```yaml
version: '3.8'

services:
  bot_api:
    build: ./bot_api
    volumes:
      - ./data:/app/data
    environment:
      - BOT_TOKEN=${BOT_TOKEN}
    restart: always

  userbot:
    build: ./userbot
    volumes:
      - ./data:/app/data
      - ./sessions:/app/sessions
    environment:
      - API_ID=${API_ID}
      - API_HASH=${API_HASH}
    restart: always
```

```bash
# Запуск
docker-compose up -d

# Логи
docker-compose logs -f
```
